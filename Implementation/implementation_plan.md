# Initial Implementation Plan

This document outlines the planned sequence of implementation tasks for the project.

## Phase 1 – Environment Setup
- Install Java, Maven, and IDE  
- Create Spring Boot skeleton  
- Configure H2 database  
- Verify application startup  

## Phase 2 – Core Structure
- Create controllers, services, and repositories  
- Set up basic routing and views  
- Add placeholder pages for vulnerable features  

## Phase 3 – Vulnerability Implementation
### Backend (Allwyn)
- SQL Injection  
- Broken Authentication  
- IDOR  

### Frontend (Thilan)
- XSS  
- CSRF  
- Insecure File Upload  

## Phase 4 – Testing & Documentation
- Test each vulnerability  
- Document exploitation steps  
- Prepare screenshots and walkthroughs  
- Update README and deliverables  

## Notes
This plan will be updated as development progresses and new tasks are identified.
